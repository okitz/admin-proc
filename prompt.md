# 抽出するアクション Type (コストの種類)
以下のTypeは、手続きに伴う「コスト」を定義したものです。

## 物理的アクション (Physical - アナログ特有のコスト)
- Physical_Go: 役所、郵便局、コンビニなどへ「移動」する、または「行く必要がある」アクション。
- Physical_Mail: ポスト投函、切手貼り、封筒の準備、郵送作業全般。
- Physical_Copy: 紙の書類を紙で「複製」する（コピー機を使う）。
- Physical_Print: PDFなどをダウンロードし「印刷」する（デジタル→アナログ変換）。
- Physical_Fill: ペンで紙に記入する、あるいは押印する。
- Physical_Attach: 書類をクリップ留め、糊付け、封筒に同封するなど、物理的な「整理・添付」作業。

## 外部依存アクション (External - 他者依存のコスト)
- External_Acquire: 会社への作成依頼、医師の診断書取得など、他者や他機関の都合で手続きが止まるアクション。

## デジタルアクション (Digital - デジタル特有のコストと作業)
- Digital_Input: フォームのテキスト項目、選択肢などを入力する、あるいは確認する。送信ボタンは含まない。
- Digital_Auth: ログイン、電子署名、マイナンバーカードのPW入力など、認証・認知的負荷の高いアクション。
- Digital_Upload: ファイルを選択し、システムにアップロードする。
- Digital_Capture: スマホやスキャナーで書類を「撮影・スキャン」する（アナログ→デジタル変換）。
- Digital_Submit: データの入力や添付が全て終わり、最後にシステムへ「送信」・「確定」するアクション。

## 時間・省略アクション (Time / Absence)
- Wait_Process: 郵送中、発行待ち、審査中など、ユーザーの手を離れて時間が経過する待機アクション。
- No_Action: マイナ連携等により、本来必要だった書類や手続きが「省略された」アクション。理想的なDXの指標。


# 抽出するCategory (主体の分類)
各アクションが「誰・何」によって行われるものか、その性質を分類するものです。すべてのステップにCategoryを付与してください。

- Work: ユーザー自身が時間をかけて行う「作業」（記入、入力、撮影、アップロード、認証）。
- Move: ユーザーの「移動」（役所へ行く、ポストへ行く）。
- Wait: システムや他者の処理を「待つ」時間（郵送、審査、発行待ち）。
